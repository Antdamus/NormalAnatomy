CARD TYPE: NORMAL HIGH-YIELD

SECTION PURPOSE

This is the default compact concept card for non-image-first normal content.

-----------------------------------------------------

TRIGGER CONDITION

NORMAL HIGH-YIELD exists only when:
- High_Yield_Q is populated

If High_Yield_Q is populated, the note must also contain:
- High_Yield_A

Question must remain blank.

If any other trigger field or front-driving prompt family is populated:
- the note is structurally invalid

-----------------------------------------------------

WHEN TO USE NORMAL HIGH-YIELD

Use it for compact, source-supported teaching points such as:
- anatomy definitions
- landmark rules
- normal appearance anchors
- approach rules
- review sequence
- normal checklist items
- measurement or morphology rules when explicitly supported
- normal variant anchors
- concise "what makes this normal" teaching points

This is the preferred home for exam-approach content when image recognition is not the primary task.

-----------------------------------------------------

FIELD MATRIX

Required:
- Clinical_Context
- High_Yield_Q
- High_Yield_A

Conditionally allowed:
- Entity_Label
- Image when a figure materially improves the concept
- Image_Annotated when annotated support is necessary
- Imaging_Differentiation
- Original_Caption when image workflow requires it
- summary when required by source hierarchy

Forbidden:
- Question
- Most_Likely_Diagnosis
- Differential_Q
- Differentials
- Mechanism_Q
- Mechanism
- Boards_Trap_Q
- Boards_Trap
- Radiopaedia_Link
- Radiopaedia_Case_Context
- Radiopaedia_Case_Summary
- Radiopaedia_Case_Differential

-----------------------------------------------------

FRONT-SIDE BLINDING

Front should contain only the fields required by the shared template for this card type.

High_Yield_Q should:
- ask for one compact learning target
- stay direct and answer-specific
- avoid embedded answers
- avoid turning into a paragraph prompt

-----------------------------------------------------

HIGH-YIELD ANSWER RULE

High_Yield_A should be compact, structured, and operational.

Good content forms:
- one-sentence definition
- short checklist
- short ordered review sequence
- compact modality anchor
- concise normal-vs-variant separator

Do NOT turn High_Yield_A into an unstructured paragraph dump.

-----------------------------------------------------

EXAM-APPROACH RULE

NORMAL HIGH-YIELD is the preferred card type for:
- landmark checklist
- normal workflow anchors
- review sequence
- what constitutes a complete normal review
- concise overcall-prevention checklist items

If the source gives a reason why the rule works:
- that reason may be better encoded separately as NORMAL MECHANISM

-----------------------------------------------------

HIGH-YIELD SAFETY CHECK

Before export, confirm:
- High_Yield_Q is populated intentionally
- High_Yield_A is populated
- High_Yield_Q is the only front-driving prompt family
- Question is blank
- no other trigger fields are populated
- no forbidden fields are populated
- Image_Annotated is blank unless Image is populated
- Original_Caption is blank unless Image is populated
- the answer is concrete and source-supported
