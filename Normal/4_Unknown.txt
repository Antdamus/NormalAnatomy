CARD TYPE: NORMAL UNKNOWN

SECTION PURPOSE

This is the primary image-recognition card type for the NORMAL engine.

-----------------------------------------------------

TRIGGER CONDITION

NORMAL UNKNOWN exists only when BOTH are populated:
- Image
- Question

If either is missing:
- this is not NORMAL UNKNOWN

If any non-UNKNOWN trigger field or front-driving prompt family is also populated:
- the note is structurally invalid

-----------------------------------------------------

WHEN TO USE NORMAL UNKNOWN

Use NORMAL UNKNOWN when the learner must identify from the image:
- a structure
- a landmark
- a spatial relationship
- an expected normal appearance
- a normal physiologic appearance
- a normal variant
- the key normal feature preventing overcall

This is NOT a disease-diagnosis card.

-----------------------------------------------------

FIELD MATRIX

Required:
- Clinical_Context
- Image
- Question

Conditionally allowed:
- Image_Annotated
- Entity_Label
- Imaging_Differentiation
- Original_Caption
- summary when required by source hierarchy

Forbidden:
- Most_Likely_Diagnosis
- Differential_Q
- Differentials
- Mechanism_Q
- Mechanism
- Boards_Trap_Q
- Boards_Trap
- High_Yield_Q
- High_Yield_A
- Radiopaedia_Link
- Radiopaedia_Case_Context
- Radiopaedia_Case_Summary
- Radiopaedia_Case_Differential

-----------------------------------------------------

QUESTION DESIGN RULE

Question should ask for one image-based recognition task only.

Good question goals:
- identify the labeled structure
- identify the landmark anchoring the review
- identify the normal relationship shown
- identify the expected normal appearance
- decide whether the finding is normal, a variant, or abnormal
- identify the feature that keeps the appearance from being overcalled

Question must NOT:
- name the answer
- embed discriminator prose that solves the card
- ask for a broad pathology differential
- become a multi-step essay prompt

-----------------------------------------------------

FRONT-SIDE BLINDING

Front should contain only the fields required by the shared template for this card type:
- Clinical_Context
- Image
- Question

Clinical_Context should remain:
- short
- neutral
- non-diagnostic
- unique at the note level

Do not leak the answer through front wording, captions, or label text.

-----------------------------------------------------

BACK-SIDE RULE

Entity_Label should hold the target answer or core concept label.

Imaging_Differentiation may briefly explain:
- why it is normal
- what makes the relationship normal
- what feature marks it as a variant rather than pathology

Do not turn the back into a long narrative.
Do not populate forbidden fields to add extra teaching.

-----------------------------------------------------

LANDMARK RULE

NORMAL UNKNOWN is an appropriate home for exam-approach content only when the approach concept is image-grounded.

Examples:
- "Identify the first landmark that should anchor review of this study."
- "What normal relationship is being confirmed on this phase or sequence?"

If the teaching point is a general review checklist rather than image recognition:
- use NORMAL HIGH-YIELD instead

-----------------------------------------------------

UNKNOWN SAFETY CHECK

Before export, confirm:
- Image is populated
- Question is populated
- Question is the only front-driving prompt family
- no non-UNKNOWN trigger fields are populated
- no forbidden fields are populated
- Image_Annotated is blank unless Image is populated
- Original_Caption is blank unless Image is populated
- the question does not leak the answer
- the image truly supports recognition
- summary is populated only when required
