SECTION PURPOSE

This file defines the explicit source hierarchy for the NORMAL engine.
It replaces any older Core-first assumption.

-----------------------------------------------------

PRIMARY SOURCE HIERARCHY FOR THE NORMAL ENGINE

Use sources in this order of authority:

1) User-provided RadPrimer article
2) STATdx
3) Core Radiology
4) Radiopaedia (last resort only)

This hierarchy is mandatory for:
- normal anatomy
- normal modality appearance
- normal variants
- pseudolesions and mimics
- exam approach
- review sequence
- sequence/phase/plane priorities

Do NOT pretend Core is the primary driver for this engine.

-----------------------------------------------------

SOURCE USAGE RULES

If a full RadPrimer article is provided:
- treat it as the primary anchor source
- derive the summary from that full article only
- do not overwrite RadPrimer-supported content with Core phrasing

If no RadPrimer article is provided but full STATdx content is provided:
- treat STATdx as the primary anchor source
- derive the summary from that full article only

Core Radiology is tertiary in this engine.
Use it only to support or backfill concepts not fully developed in the primary article, and only when this does not conflict with the higher-priority source.

Core content is allowed only when it is actually available in the current run. Valid Core evidence means exact Core text/pages in the prompt, exact user-provided Core excerpts/pages, or retrievable ChatGPT project/source-file content from an uploaded Core Radiology PDF or equivalent source. A note such as "look through Core" is permission to search, not source support.

If the assistant independently finds the relevant Core pages/section inside the available ChatGPT project/source files, it may use that Core content without asking the user for page numbers. It must still state the retrieved Core source basis when visible.

If Core cannot be retrieved or quoted in the current run:
- do not add Core-derived facts
- do not label any card or summary as Core-supported
- keep RadPrimer/STATdx-derived summaries article-only
- proceed with the higher-priority available source rather than fabricating Core coverage

Radiopaedia is last resort only.
Use it only when the needed normal concept is absent from higher-priority sources and the concept is narrow, stable, and clearly described.

-----------------------------------------------------

SOURCE CONFLICT RULE

If allowed sources differ, resolve by hierarchy:
- RadPrimer overrides STATdx, Core, and Radiopaedia
- STATdx overrides Core and Radiopaedia
- Core overrides Radiopaedia

Do not silently blend conflicting claims into one smoothed statement.
If the conflict matters and cannot be resolved cleanly, omit the disputed claim.

-----------------------------------------------------

OUTSIDE LITERATURE RULE

Outside literature remains forbidden by default.

Narrow exceptions are allowed only for:
- stable quantitative normal facts that are truly necessary and absent from the allowed hierarchy
- stable diagnostic-performance facts when the source raises a performance question but gives incomplete details
- mechanism, developmental, physiologic, histologic, sequence/phase, or modality-physics clarification when the anchor source states the appearance or discriminator but does not clearly explain why it occurs

External literature may clarify why an article-supported pattern occurs.
It must not introduce new standalone anatomy facts, variants, workflows, interpretation rules, or disease claims.

When outside literature is used in a card back:
- label it explicitly inside the same field as: "External literature source (not stated in RadPrimer/STATdx/Core/Radiopaedia):"
- include the exact source link in the same field
- keep the article-supported fact and the external explanation distinguishable
- do not add the outside content to the article-level summary

-----------------------------------------------------

SUMMARY DISCIPLINE RULE

For RadPrimer- or STATdx-derived cards:
- summary is required
- summary must be derived only from the full anchor article
- do not mix in Core
- do not mix in Radiopaedia
- do not add outside knowledge

For Core-derived cards without RadPrimer/STATdx anchor content:
- summary must remain blank

-----------------------------------------------------

FULL-SOURCE REVIEW RULE

Do not generate cards from:
- memory of a RadPrimer article
- excerpts only
- cached digest text
- prior summaries

The engine must behave as a full-article extraction workflow, not a recollection workflow.
