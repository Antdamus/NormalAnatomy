CARD TYPE: NORMAL BOARDS TRAP

SECTION PURPOSE

This card type captures normal findings that are commonly overcalled or misread.

-----------------------------------------------------

TRIGGER CONDITION

NORMAL BOARDS TRAP exists only when:
- Boards_Trap is populated

If Boards_Trap is populated, the note must also contain:
- Boards_Trap_Q
- Entity_Label

Question must remain blank.

If any other trigger field or front-driving prompt family is populated:
- the note is structurally invalid

-----------------------------------------------------

WHEN TO USE NORMAL BOARDS TRAP

Use NORMAL BOARDS TRAP for:
- normal mimics of disease
- pseudolesions
- variants mistaken for pathology
- structures commonly overcalled
- modality-specific normal misreads
- omission traps in reviewing a normal exam

This is a high-value card type in the NORMAL engine.

-----------------------------------------------------

FIELD MATRIX

Required:
- Clinical_Context
- Entity_Label
- Boards_Trap_Q
- Boards_Trap

Conditionally allowed:
- Image when the trap is inherently visual
- Image_Annotated when annotated support exists
- Imaging_Differentiation
- Original_Caption when non-annotated image display needs a caption
- Most_Likely_Diagnosis only as diagnosis-only anchor if truly needed
- summary when required by source hierarchy

Forbidden:
- Question
- Differential_Q
- Differentials
- Mechanism_Q
- Mechanism
- High_Yield_Q
- High_Yield_A
- Radiopaedia_Link
- Radiopaedia_Case_Context
- Radiopaedia_Case_Summary
- Radiopaedia_Case_Differential

-----------------------------------------------------

FRONT-SIDE BLINDING

Front may contain only:
- Clinical_Context
- Boards_Trap_Q
- Image if the trap is visual

Front must NOT contain:
- answer leakage
- differential lists
- caption-derived answer prose
- named diagnoses unless a tightly bounded neutral anchor is unavoidable

-----------------------------------------------------

TRAP CONTENT RULE

Boards_Trap must encode:
- what the trap is
- what feature, check, maneuver, or landmark keeps the reader safe
- the key discriminator that resolves the ambiguity
- why the normal structure may be mistaken for a lesion
- whether it is a normal structure, variant, or pseudolesion
- how to verify the expected location using visible landmarks

Do not stop at naming the trap.

-----------------------------------------------------

OVERCALL-PREVENTION RULE

Imaging_Differentiation is strongly encouraged for this card type.
Use it to state the separator that prevents misdiagnosis.

Examples of valid separator categories:
- preserved normal anatomy
- lack of aggressive behavior
- expected course or attachment
- phase or sequence consistency
- stability of a normal interface
- expected ventricular, cisternal, peduncular, or compartmental position

-----------------------------------------------------

TRAP SAFETY CHECK

Before export, confirm:
- Boards_Trap is populated intentionally
- Boards_Trap_Q is populated
- Boards_Trap_Q is the only front-driving prompt family
- Entity_Label is populated
- Question is blank
- no other trigger fields are populated
- no forbidden fields are populated
- Image_Annotated is blank unless Image is populated
- Original_Caption is blank unless Image is populated
- the trap is specific and source-supported
