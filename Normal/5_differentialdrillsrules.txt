CARD TYPE: NORMAL DIFFERENTIAL DRILL

SECTION PURPOSE

This card type teaches true discrimination.
It is not the default output for normal-anatomy material.

-----------------------------------------------------

TRIGGER CONDITION

NORMAL DIFFERENTIAL DRILL exists only when:
- Differentials is populated

If Differentials is populated, the note must also contain:
- Differential_Q
- Imaging_Differentiation

Question must remain blank.

If any other trigger field or front-driving prompt family is populated:
- the note is structurally invalid

-----------------------------------------------------

WHEN TO USE NORMAL DIFFERENTIAL DRILL

Use it only when the source teaches a real separator between:
- normal and abnormal
- normal variant and pathology
- pseudolesion and true lesion
- two closely related mimic entities when the discriminator is the key teaching point

Do NOT use it for routine anatomy naming.
Do NOT use it just because comparison feels educational in the abstract.

-----------------------------------------------------

FIELD MATRIX

Required:
- Clinical_Context
- Differential_Q
- Differentials
- Imaging_Differentiation

Conditionally allowed:
- Image when the discrimination task is image-grounded
- Image_Annotated when paired image support is needed
- Entity_Label
- Most_Likely_Diagnosis only as diagnosis-only anchor if truly needed
- Original_Caption only when image workflow requires it
- summary when required by source hierarchy

Forbidden:
- Question
- Mechanism_Q
- Mechanism
- Boards_Trap_Q
- Boards_Trap
- High_Yield_Q
- High_Yield_A
- Radiopaedia_Link
- Radiopaedia_Case_Context
- Radiopaedia_Case_Summary
- Radiopaedia_Case_Differential

-----------------------------------------------------

FRONT-SIDE BLINDING

Front may contain only:
- Clinical_Context
- Differential_Q
- Image stack if present

Front must NOT contain:
- diagnosis names
- differential names
- named signs
- caption-derived answer phrases
- interpretive prose that gives away the separator

-----------------------------------------------------

DIFFERENTIAL CONTENT RULE

The Differentials field must stay tightly bounded.

Include only realistic competitors that the source actually teaches for the card's purpose.
Do NOT inflate into a pathology-deck style list.

Good uses:
- variant vs mass
- pseudolesion vs true lesion
- normal developmental appearance vs abnormal finding

Bad uses:
- broad disease catalogues
- speculative alternatives
- filler entries added to make the field non-empty

Comparison cards should survive only when they sharpen judgment in a way that will help the learner read studies better.

-----------------------------------------------------

DISCRIMINATOR RULE

Imaging_Differentiation must teach the dominant separator.

Use it for the key discriminator, such as:
- preserved normal relationship
- expected enhancement pattern
- expected vessel course
- lack of mass effect
- expected sequence behavior
- stable landmark position

If there is no meaningful discriminator:
- do not create this card

-----------------------------------------------------

DIFFERENTIAL SAFETY CHECK

Before export, confirm:
- Differentials is populated intentionally
- Differential_Q is populated
- Differential_Q is the only front-driving prompt family
- Imaging_Differentiation is populated
- Question is blank
- no other trigger fields are populated
- no forbidden fields are populated
- Image_Annotated is blank unless Image is populated
- Original_Caption is blank unless Image is populated
- the front does not leak the answer
