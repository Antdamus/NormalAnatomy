SECTION PURPOSE

This file transfers the pathology engine's architectural discipline into the NORMAL engine without importing pathology semantics.

It defines:
- required fields
- conditionally allowed fields
- normally blank fields
- forbidden fields
- trigger dependencies
- anti-dual-trigger rules

Blank is better than accidental triggering.

-----------------------------------------------------

GLOBAL MATRIX RULE

For any single note, populate only the fields needed for the one intended card type.

If a trigger field for another card type is populated by accident, the note is structurally invalid.

Trigger fields are:
- Question together with Image -> NORMAL UNKNOWN
- Differentials -> NORMAL DIFFERENTIAL DRILL
- Mechanism -> NORMAL MECHANISM
- Boards_Trap -> NORMAL BOARDS TRAP
- High_Yield_Q -> NORMAL HIGH-YIELD

-----------------------------------------------------

GLOBAL FORBIDDEN FILLER RULE

Never populate any field with:
- filler
- N/A
- none
- not applicable
- pseudo-diagnoses
- fake differentials
- pseudo-mechanisms
- unsupported claims
- duplicated content inserted only to avoid blanks

If a field is not needed, leave it blank.

-----------------------------------------------------

NORMAL UNKNOWN - FIELD MATRIX

Required:
- Clinical_Context
- Image
- Question

Conditionally allowed:
- Image_Annotated
- Entity_Label
- Imaging_Differentiation
- Original_Caption
- summary when required by source hierarchy

Normally blank:
- Most_Likely_Diagnosis
- Differential_Q
- Mechanism_Q
- Boards_Trap_Q
- High_Yield_A

Forbidden:
- Differentials
- Mechanism
- Boards_Trap
- High_Yield_Q
- Radiopaedia_Link
- Radiopaedia_Case_Context
- Radiopaedia_Case_Summary
- Radiopaedia_Case_Differential

Isolation rule:
- If Question is populated for an image card, the note is NORMAL UNKNOWN and must not populate another trigger field.

-----------------------------------------------------

IMAGE / QUESTION COUPLING RULE

Question may be populated only for NORMAL UNKNOWN.

Therefore:
- Image + Question = NORMAL UNKNOWN
- Image without Question may support another card type
- Any non-NORMAL-UNKNOWN note with Image must keep Question blank

If Image and a non-UNKNOWN trigger field coexist while Question is also populated:
- the note is structurally invalid

-----------------------------------------------------

DUAL-TRIGGER BLOCKER

The following combinations are forbidden on one note:
- Differentials + Mechanism
- Differentials + Boards_Trap
- Differentials + High_Yield_Q
- Mechanism + Boards_Trap
- Mechanism + High_Yield_Q
- Boards_Trap + High_Yield_Q
- Question + any non-UNKNOWN trigger field

If any forbidden combination appears:
- clear the unintended field(s)
OR
- split into separate notes

Do not export a dual-trigger note.
