CAPTION VERBATIM ENFORCEMENT (ABSOLUTE — ZERO MODIFICATION POLICY)

Any caption placed in:
• Image_Annotated stackCap
• Original_Caption

MUST be an EXACT character-for-character match to the user-provided caption text.

This includes preserving exactly:

• Capitalization
• Punctuation
• Quotation marks (straight vs curly)
• Parentheses
• Hyphenation
• Figure labels (e.g., (a–d))
• Line-internal spacing
• En dashes vs hyphens
• All numeric formatting
• Abbreviations exactly as written

The assistant MUST NOT:

• Paraphrase
• “Clean up” grammar
• Normalize quotation marks
• Convert hyphens/dashes
• Remove figure labels
• Remove clauses
• Reorder clauses
• Re-wrap lines
• Insert interpretation
• Correct perceived typos
• Standardize spacing
• Modify even a single character

If the provided caption text cannot be preserved exactly as given:
• STOP immediately.
• Request the caption text again.
• Do NOT attempt reconstruction from memory.
• Do NOT approximate.

SELECTED IMAGE / CASE COVERAGE AUDIT

Before TSV export, account for every selected image block or explicit CASE block supplied in the input.

For each selected image or grouped case, mark it as one of:
- carded as a primary image-recognition case
- used as support inside another source-supported card
- intentionally omitted as redundant or non-contributory

Do not silently skip a selected image or case. If a selected image is a teaching graphic, flowchart, site map, gross/pathology photo, normal comparator, modality comparator, or pitfall image, it may deserve a high-yield or mechanism-style card even when it is not a classic "most likely diagnosis" case.

If any selected image/case is unaccounted for, stop and repair the deck before TSV export.

  “TSV export blocked — caption verbatim rule violated.”
