HIGH-YIELD CARD RULES

Card Type: HIGH-YIELD

Trigger Condition:
• HIGH-YIELD card is generated when the High_Yield_Q field is populated.

Fields Used:
• High_Yield_Q
• High_Yield_A

All other fields must remain blank unless intentionally generating another card type.

-----------------------------------------------------

Purpose of HIGH-YIELD Cards:

High-Yield cards exist to encode:

• Numeric thresholds
• Size cutoffs
• Management triggers
• Staging rules
• Classification pivots
• “Most common…” statements
• Epidemiologic pivots that change interpretation
• Board-relevant discriminators
• Exam-critical decision rules

ONTOLOGY LIMITATION (DEFINITION-ONLY)

High-Yield fields MUST NOT be used for Ontology Spine cards.

Ontology encoding is DEFINITION-ONLY and must be implemented ONLY via:
• High_Yield_Q
• High_Yield_A
with Tags including: Ontology::Definition

Do NOT create:
• Ontology classification cards
• Ontology discriminator cards
• Any multi-card “ontology spine”

High-Yield fields remain reserved for true board pivots
(thresholds, staging, management triggers, classification pivots, “most common” pivots),
EXCEPT the single allowed Definition card per new entity.
-----------------------------------------------------

SYNDROME ENCODING RULE (MANDATORY — HIGH-YIELD):

If the topic/entity is a named syndrome (e.g., a classic triad/pentad pattern, or a board-known syndrome label):

If Sectioned Build mode is active, these syndrome cards must be generated ONLY in SECTION G.

You MUST generate TWO separate High-Yield cards, both source-supported:

CARD 1 — “Syndrome → Typical Presentation”
• High_Yield_Q: “For [SYNDROME NAME], what is the typical presentation (classic features)?”
• High_Yield_A: Concise list of the classic features (triad/pentad/key elements), exactly as supported by Core/RadPrimer/STATdx.
• Do NOT add extra features not explicitly supported.

CARD 2 — “Typical Presentation → Syndrome”
• High_Yield_Q: “Named syndrome: [classic feature list]. What is the syndrome?”
  - The feature list must be short and canonical (the minimum set that uniquely cues the syndrome),
    and must be explicitly supported by Core/RadPrimer/STATdx.
• High_Yield_A: “[SYNDROME NAME]” (plus a one-line anchor if needed, but keep it tight).

Trigger / Output Notes:
• Each of the two cards must be its own note line (not combined into one field).
• This rule applies whether the High-Yield cards are case-bundled or standalone.
• If standalone: Clinical_Context MUST be the required 12-character alphanumeric/symbolic code (per your existing standalone rule).
• Do NOT create syndrome cards if the syndrome pattern is not explicitly stated in allowed sources.
  (No inference-based syndrome creation.)
  
They are NOT for:
• Long explanations
• Mechanisms (unless threshold-related)
• Imaging pattern description
• Differential lists
• Trivia
• Low-impact epidemiology
• Reworded textbook sentences

-----------------------------------------------------

High-Yield Generation Modes:

High-Yield cards may be generated in two ways:

1) Case-Bundled High-Yield  
   • Generated alongside an image case or entity case.  
   • Clinical_Context follows the case’s normal structure.  

2) Standalone High-Yield (Concept-Only)  
   • Generated in isolation (no associated case).  
   • Clinical_Context MUST contain a unique 12-character alphanumeric/symbolic code.  
   • Clinical_Context must contain NOTHING else.  
   • If this 12-character code is missing, the card will NOT be generated by anki.

-----------------------------------------------------

Front-Side Rules:

Front contains ONLY:
• High_Yield_Q

No:
• Diagnosis labels
• Imaging findings
• Context hints
• Case descriptions
• Modality references unless threshold-specific
• Embedded answers

The question must be direct, precise, and testable.

-----------------------------------------------------

Back-Side Rules:

Back contains ONLY:
• High_Yield_A

The answer must be:

• Concise
• Board-relevant
• Explicit
• Free of speculation
• Free of invented data

If numeric:
• Numbers must be source-supported
• No fabricated percentages

If epidemiology:
• Must alter interpretation
• Must be explicitly supported by allowed sources

-----------------------------------------------------

EPIDEMIOLOGY QUALITY FLOOR (HIGH-YIELD ONLY — EXAM PIVOT REQUIRED)

Epidemiology facts may be encoded ONLY when they change interpretation or test-taking behavior.

Every Epidemiology High-Yield card MUST:

• State the exam-relevant pivot explicitly (WHY this epi fact matters).
• Use only source-stated numbers/claims (no rounding, no extrapolation, no combining sources).
• Prefer the smallest sufficient statement (1–3 lines).
• Avoid vague phrasing (“common”, “often”, “rare”) unless the source uses exactly that wording.

Allowed Epidemiology pivots include ONLY:
• “Most common …” statements that change ranking in a differential.
• Age/sex distribution that materially narrows the diagnosis.
• Strong association statements explicitly stated in the source that change your diagnostic prior.
• Source-stated proportion/prevalence facts ONLY when they materially change diagnostic ranking or default assumptions.

Forbidden:
• Minor prevalence trivia
• Geographic trivia
• Risk factors that do not change imaging interpretation
• Any epidemiology that is not explicitly stated in Core/RadPrimer/STATdx

EPIDEMIOLOGY QUESTION SPECIFICITY RULE (MANDATORY)

Epidemiology High-Yield questions must ask for the SPECIFIC source-supported fact being tested.

They must NOT use generic stems such as:
• “What is the single exam-relevant epidemiology fact?”
• “What is the key epidemiology fact?”
• “What is the epidemiology pivot?”

Instead, High_Yield_Q MUST explicitly ask for the precise item, such as:
• prevalence / proportion
• age distribution
• sex distribution
• most common location
• most common cause
• familial association
• ethnic distribution
• other source-supported epidemiologic ranking fact

For Epidemiology High-Yield cards, the front must be answer-specific.
A reader should be able to tell exactly what fact is being requested before flipping the card.

Preferred examples:
• “What percentage of pancreatic malignancies are pancreatic ductal adenocarcinomas?”
• “What is the median age at onset for pancreatic ductal adenocarcinoma?”
• “What is the most common location of pancreatic ductal adenocarcinoma within the pancreas?”
• “What proportion of patients with pancreatic ductal adenocarcinoma have a first-degree relative with pancreatic cancer?”
• “Which ethnic group has a higher incidence of pancreatic ductal adenocarcinoma according to the source?”

Required HY formatting for Epidemiology cards:

High_Yield_Q:
• Must ask for the exact epidemiologic fact being tested in concrete terms.
• Must be direct, precise, and testable.
• Must NOT be generic or umbrella-worded.

High_Yield_A (must include BOTH):
1) The fact (verbatim or tightly faithful to source)
2) The pivot line: “Exam pivot: …” (why it matters)

If the “Exam pivot” cannot be stated in one line:
• DO NOT generate the epidemiology card.

-----------------------------------------------------

MRI APPEARANCE ENCODING RULE (MANDATORY WHEN SOURCE SUPPORTS)

If the provided Core/RadPrimer/STATdx source explicitly describes MRI appearance
(signal characteristics, enhancement pattern, diffusion, or named MRI signs):

You MUST generate at least ONE dedicated High-Yield card encoding that MRI appearance.

(If Sectioned Build mode is active, this requirement is QUEUED and must be emitted only in SECTION G.)

Rules:
• MRI details may be included ONLY if explicitly stated in the source.
• Include the sequence name(s) exactly as written (e.g., T1, T2, DWI, post-contrast).
• Do NOT infer unmentioned sequences or signal characteristics.
• Do NOT generalize beyond the specific source statement.
• Keep the card tight: 2–6 short lines max.

Recommended format:

High_Yield_Q:
“MRI appearance pivot for [ENTITY]: what is the key MRI signal/enhancement pattern (as stated in source)?”

High_Yield_A:
• Sequence: [source-stated sequence] → [source-stated appearance]
• (repeat for other sequences only if source states them)
• Exam pivot: [one line: why this helps discriminate or recognize]

If the MRI description is too long:
• Split into multiple High-Yield cards ONLY if each card is its own distinct pivot.
• Otherwise omit lower-value details.
-----------------------------------------------------

Inflation Rule:

Do NOT generate high-yield cards for:

• Redundant facts
• Minor epidemiology
• Non-decision-impacting details
• Restatements of image findings
• Content already encoded in another high-yield card

Each high-yield card must represent a distinct board pivot.

-----------------------------------------------------

Integrity Rule:

High-Yield cards must NOT:

• Duplicate content from Mechanism cards
• Duplicate content from Differential Drill logic
• Leak diagnosis-specific pattern recognition
• Expand beyond what is explicitly supported in Core / RadPrimer / STATdx / Radiopaedia

Quality over quantity.

-----------------------------------------------------
DIAGNOSTIC PERFORMANCE & THRESHOLD RULE
(MUTUAL RECALL + SOURCE TRANSPARENCY + ACTIVE REVIEW REQUIREMENT)

This applies to High-Yield cards that encode:

• Sensitivity
• Specificity
• PPV / NPV
• Accuracy
• Likelihood ratios
• HU cutoffs
• SUV thresholds
• Size thresholds
• Enhancement thresholds
• Quantitative staging criteria
• Any numeric diagnostic performance metric
• Comparative lab-test performance when board-relevant
• Best-test performance in a specific clinical setting when board-relevant

-----------------------------------------------------

0) ACTIVE REVIEW REQUIREMENT (MANDATORY)

During SECTION G, the assistant MUST explicitly review whether the topic has
diagnostic performance data worth encoding.

This review must include, when relevant:

• Imaging test sensitivity / specificity
• Comparative performance of one modality versus another
• Lab-test performance when it materially affects diagnosis
• “Best test” or “most useful test” performance in a specific clinical setting
• Quantitative thresholds that change interpretation

If such data are:
• explicitly present in Core / RadPrimer / STATdx / Radiopaedia
  → encode them in SECTION G
• absent from those sources but likely high-yield and clinically important
  → the assistant SHOULD consider external literature under the external-literature exception
• absent, weak, highly heterogeneous, conflicting, or not exam-relevant
  → do NOT force generation

The goal is to capture stable, high-impact diagnostic performance pivots,
not to accumulate low-value statistics.

-----------------------------------------------------

1) MUTUAL RECALL STRUCTURE (ALLOWED & PREFERRED)

If multiple performance metrics are tightly linked
(e.g., sensitivity + specificity),
they MAY be encoded on the SAME card
to enforce mutual recall.

Example format:

High_Yield_Q:
“What are the sensitivity and specificity of CT for X?”

High_Yield_A:
• Sensitivity: ___
• Specificity: ___
• Exam pivot: ___

This is preferred when:
• The metrics are commonly tested together
• The pairing improves recall strength
• Both metrics are equally exam-relevant

-----------------------------------------------------

2) SOURCE PRIORITY

Primary source hierarchy still applies:

1) Core Radiology
2) RadPrimer
3) STATdx
4) Radiopaedia

If performance metrics are NOT provided in those sources:

You MAY include externally sourced values
ONLY if:

• They come from a credible medical source
  (major review, textbook, guideline, meta-analysis).
• They are widely accepted values (not single small studies).
• They are stable enough to improve test-taking or interpretation.
• They are clearly labeled as outside-source.

In that case, you MUST include the following lines inside High_Yield_A:

“External literature source (not stated in Core/RadPrimer/STATdx/Radiopaedia).”
“Source: [stable link]”

Both lines must appear explicitly in High_Yield_A.
Silent external sourcing is prohibited.

The source link must point to the exact article, guideline, review, or reference supporting the card.
Acceptable links include:
• DOI URL
• PubMed URL
• journal article landing page
• guideline page

If multiple sources were reviewed, use the single most directly supporting source for the final wording of the card.

If no stable source link can be provided:
• Do NOT generate the externally sourced High-Yield card.

-----------------------------------------------------

3) EXACTNESS RULE

• Preserve numbers exactly as written in the source.
• Preserve ranges exactly.
• Preserve “approximately” wording if used.
• Do NOT round.
• Do NOT merge multiple studies.
• Do NOT average values.

If multiple external sources conflict:
• Do NOT generate the card.

-----------------------------------------------------

4) EXAM PIVOT REQUIREMENT

Every performance card MUST include:

• One short line:
  “Exam pivot: ____.”

This must explain how the number changes test-taking or interpretation.

If no pivot exists:
• Do NOT generate the card.

-----------------------------------------------------

5) INFLATION CONTROL

Do NOT generate performance cards when:

• Numbers are vague or weakly stated.
• The metric does not influence interpretation.
• The number is trivial or rarely tested.
• The performance varies so widely that no stable pivot exists.
• The metric is too technique-dependent to support a durable board-style card.

Performance cards exist to encode high-impact diagnostic metrics,
not to accumulate statistics.

-----------------------------------------------------

6) SECTION G PERFORMANCE SUBBLOCK REQUIREMENT

Within SECTION G, diagnostic performance cards should be treated as an explicit subblock
whenever the source review or permitted external review yields stable, board-relevant values.

These cards remain part of SECTION G.
Do NOT create a separate section for performance unless the user explicitly requests one.
