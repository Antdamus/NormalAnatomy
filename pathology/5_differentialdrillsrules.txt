# DIFFERENTIAL DRILL — PATTERN-BASED ARCHITECTURE (core_rad_notetype)

## PURPOSE

Differential Drill is a SEPARATE card type from UNKNOWN.

It exists to force:

• Active differential generation  
• Structured A-vs-B-vs-C discrimination  
• Dominant imaging discriminator recognition  
• Board-level cognitive pivoting  

It is a discrimination engine — NOT a diagnosis card.

---

# CARD-TYPE TRIGGER RULE (STRICT)

Differential Drill is generated ONLY when:

• `Differentials` field is populated.

Therefore:

• NEVER populate `Differentials` unless intentionally generating a Differential Drill.

If `Differentials` is populated:

• `Differential_Q` MUST be populated  
• `Imaging_Differentiation` MUST be populated  
• `Question` field MUST remain blank  

Failure of any = structural error.

---

# FIELD USAGE (DIFFERENTIAL DRILL)

## Required (when triggered)

• Clinical_Context  
• Differential_Q  
• Differentials  (TRIGGER FIELD)  
• Imaging_Differentiation  

## Conditionally Required

• Image (REQUIRED if case-derived; see Mode 1)

• Entity_Label

## MUST REMAIN BLANK

• Question

• Mechanism_Q  
• Mechanism

• Boards_Trap

• High_Yield_A  

Differential Drill must remain structurally isolated.

---

# FRONT-SIDE BLINDING (STRICT — ZERO LEAKAGE)

Front may contain ONLY:

• Clinical_Context  
• Differential_Q  
• Image stack (if present)  

Front must NOT contain:

• Diagnosis names  
• Differential names  
• Named signs  
• Caption-derived keywords  
• Management cues  
• Etiologic adjectives  
• Imaging pattern descriptors  
• Any narrowing contextual clues  

Zero leakage tolerance.

---

# CLINICAL_CONTEXT RULE (MODE 1 + MODE 2)

Clinical_Context must be:

• Neutral  
• Non-diagnostic  
• Clinically realistic when case-derived  
• Unique at the note level  

Clinical_Context must NOT include:

• Target diagnosis  
• Near-diagnostic disease labels  
• Etiologic adjectives implying the answer  
• Named signs  
• Imaging descriptors  

If case-derived (Mode 1):

• MUST preserve the same blinded clinical wording as the associated UNKNOWN card
• MUST remain non-diagnostic under the UNKNOWN Clinical_Context standard
• MUST be made unique at the note level to prevent duplicate-note conflicts

Therefore, Mode 1 Clinical_Context must use:

[associated UNKNOWN Clinical_Context] + [unique drill suffix]

Allowed suffix examples:
• [DD1]
• [DD2]
• ::DD1
• •DD1

The uniqueness suffix:
• Must be short
• Must not add medical meaning
• Must not reveal diagnosis
• Must be deterministic within the build when possible

Example:
• UNKNOWN: “64-year-old woman with abdominal pain and fever”
• Differential Drill: “64-year-old woman with abdominal pain and fever [DD1]”

If standalone (Mode 2):

• Must be a unique 12-character alphanumeric/symbolic ID  
• Must contain NOTHING else  

---

# DIFFERENTIAL_Q — STRICT CONTENT RULES

Differential_Q must NEVER include:

• Diagnosis names  
• Differential names  
• Named signs  
• Imaging descriptors  
• Pattern descriptions  
• Etiologic hints  
• Partial answer cues  

Allowed structure:

• Optional modality label  
• Generic instruction  
• “Name N key differentials…” (2–4 only)  
• “and give ONE best discriminator for each.”  

Safe Example:

“CT abdomen with contrast. Based on the image, name 3 key differentials and give one best discriminator for each.”

If Differential_Q contains a description of the pattern being tested  
(e.g., “peritoneal thickening with ascites”)  
→ this is considered diagnostic leakage  
→ rewrite before TSV.

---

# DIFFERENTIALS (TRIGGER FIELD)

• Must be source-supported OR explicitly labeled as model-inferred per CONTROLLED INFERENCE
• If model-inferred: must include the label “Model-inferred differentials (not listed in Core/RadPrimer/STATdx; generated by ChatGPT from caption-described imaging pattern).”
• Must reflect realistic mimics  
• 2–4 items only  
• No encyclopedic lists  
• No speculative additions  

Do NOT populate unless intentionally generating a drill.

---

# Imaging_Differentiation (REQUIRED)

Must:

• Provide workstation-grade A vs B vs C logic  
• Focus on the primary discriminator  
• Tie discriminators to:
  - Caption-derived findings (if case-derived)
  - Explicit source facts  
• Avoid vague prose  
• Avoid invented findings  
• Reflect full-source review, not digest-only reasoning

If Differentials were model-inferred (not source-listed):
• Imaging_Differentiation must rely ONLY on caption-described or image-observable discriminators.
• Do NOT use epidemiology, lab values, management triggers, or MRI sequence behavior unless explicitly source-stated.

If drill differs from another pattern drill, begin with:

“Pattern differs because: [dominant discriminator reason].”

---

# PATTERN-BASED DRILL GENERATION

Drills are generated by:

Dominant Imaging Pattern Category  
NOT identical differential lists.

---

## STEP 1 — Identify Dominant Pattern

Before generating drills, classify each UNKNOWN case by its primary discriminator.

Examples:

• Calcified/Encasing Peritoneum Pattern  
• Smooth Diffuse Enhancing Peritoneum Pattern  
• Gas-Containing Ascites Pattern  
• Post-Operative Leak Pattern  
• Loculated Collection Pattern  
• Solid Polypoid Mass Pattern  
• Circumferential Stricture Pattern  

Pattern classification is mandatory.

---

## STEP 2 — One Drill Per Distinct Pattern

Generate:

• ONE drill per dominant imaging pattern.

Do NOT generate multiple drills for cases that share:

• Same primary discriminator  
• Same separation logic  

Even if differential lists vary slightly.

---

## STEP 3 — Redundancy Suppression

Do NOT generate a new drill if:

• A-vs-B-vs-C logic is materially identical  
• Same mental separation strategy would be repeated  

Dominant discriminator equality overrides list variation.

---

## STEP 4 — Cognitive Optimization Rule

A new drill is allowed ONLY if it:

• Trains a distinct primary discriminator  
• Forces a different separation strategy  
• Improves discrimination capacity  

If not → do NOT generate.

---

# MODE 1 — CASE-BUNDLED DIFFERENTIAL DRILL (DEFAULT)

If derived from an existing UNKNOWN case:

• MUST generate as a separate note  
• MUST preserve the same blinded clinical wording as the associated UNKNOWN  
• MUST use a unique Clinical_Context suffix to prevent duplicate-note conflicts  
• MUST reuse EXACT SAME Image field  
• MUST reuse EXACT SAME Image_Annotated stack (if present)  

Image attachment is mandatory.

Standalone conversion prohibited.

---

## Entity_Label (Mode 1)

• Recommended  
• Diagnosis only  
• No descriptors  
• No findings  
• No reasoning  

(Back anchor only — never relied upon for front.)

---

## Most_Likely_Diagnosis (Mode 1)

• Blank OR diagnosis-only (single line)  
• NEVER include:
  - Caption keywords  
  - Reasoning  
  - Differential comparisons  

Those belong ONLY to UNKNOWN cards.

---

# MODE 2 — STANDALONE DIFFERENTIAL DRILL (RARE)

Allowed ONLY if:

• No specific image case exists  
OR  
• Intentionally building concept-level discrimination engine  

Requirements:

• Clinical_Context = unique 12-character ID  
• Entity_Label = REQUIRED  
• Image = optional  
• Differential_Q = REQUIRED  
• Most_Likely_Diagnosis = blank OR diagnosis-only  

Question field MUST remain blank.

---

# STRUCTURAL PRIORITY RULE

If drill originates from an existing UNKNOWN case:

→ MUST use MODE 1  
→ Standalone mode prohibited  

This preserves:

• Image coupling  
• Context integrity  
• Cognitive layering  

---

# SUMMARY FIELD RULE

If drill originates from RadPrimer/STATdx:

• summary MUST be populated  
• Must follow the global Summary Field Rule  
• Must be derived from full-article review  
• Must be article-only  
• Must contain no Core mixing  
• Must contain no inference  

Otherwise leave blank.
