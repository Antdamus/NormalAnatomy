# Guided Image Mastery Narrative Prompt

You are transforming a radiology source article into a **GUIDED IMAGE MASTERY NARRATIVE** for a radiology resident.

This output is designed for **fast, high-yield first-pass learning** when time is limited.

It is **not** primarily a flashcard replacement.
It is **not** a trivia quiz stream.
It is **not** a generic summary.
It is **not** entertainment-only storytelling.

The narrative must do four things at once:

1. **Reorganize dry article content into a coherent teaching sequence**
2. **Force deliberate inspection of the extracted images**
3. **Preserve the important educational payload of the article**
4. **Build a strong conceptual framework that Anki can later reinforce**

---

## PRIMARY EDUCATIONAL ROLE

This guided narrative is the **understanding + visual organization layer**.

The main recall burden should still remain in **Anki**.

Therefore:

* The narrative should include **targeted pauses and brief prompts**
* But it should **not** become a full barrage of detailed recall questions
* The narrative should be optimized for:

  * comprehension
  * pattern organization
  * visual attention
  * discriminative noticing
  * case-to-case comparison
  * retention of major pivots

Think of this as:

**Narrative = guided understanding + image walkthrough + structure**
**Anki = deeper retrieval practice + durable memory consolidation**

---

## INPUT FORMAT EXPECTATION

You will receive an extracted block that may contain sections such as:

* `=== TOPIC ===`
* `=== CORE VALIDATION INPUT ===`
* `=== PROMPT ===`
* `=== ARTICLE ===`
* `=== IMAGES (selected; original numbering preserved) ===`

The article section contains the structured content outline.
The image section contains the extracted image labels, filenames, and captions.

Possible image labels:

* `IMAGE_01`, `IMAGE_02`, etc.
* `CASE_01`, `CASE_02`, etc.

These labels do **not** always mean the same thing.

### IMPORTANT IMAGE / CASE INTERPRETATION RULE

If the extractor output uses `CASE_##` because explicit grouping was provided, you may treat that as a true grouped case.

If the extractor output uses `IMAGE_##`, you must **not** assume those are already true cases.

However, you may attempt **controlled grouping inference** from the captions.

---

## CONTROLLED GROUPING INFERENCE RULE

When preparing the guided narrative, you may infer likely multi-image case groupings from caption language such as:

* “in the same patient”
* “same patient”
* “same case”
* modality progression across sequential images
* direct cross-reference to a prior image
* explicit continuation language

### Grouping rules

1. If grouping confidence is **high**, you may narrate the images together as a likely grouped case.
2. If grouping confidence is **moderate**, you must explicitly label the grouping as **inferred**.
3. If grouping confidence is **low**, keep the images separate.
4. Do **not** state inferred grouping as certain unless the extractor explicitly grouped it.
5. When uncertain, prefer language like:

   * “This image likely pairs with the prior image.”
   * “These next two images appear to belong to the same patient.”
   * “This is likely an inferred multi-image case.”
6. Do **not** silently merge many images into a single case without saying so.

### Transparency rule

If you group images by inference, you must explicitly acknowledge that the grouping is inferred from caption language.
This should normally happen once near the beginning of the narrative, not as a repeated local disclaimer.

---

## GLOBAL GROUPING TRANSPARENCY RULE

If any multi-image groupings are inferred from caption continuity rather than explicitly provided by the extractor:

* acknowledge that once near the beginning of the narrative in a compact orientation note
* after that, proceed naturally with the grouped walkthrough
* re-mention uncertainty locally only if a particular grouping is especially weak or ambiguous
* do not repeatedly insert the same inferred-grouping disclaimer throughout the narrative
* avoid repetitive phrasing such as:

  * “these likely belong together”
  * “this is an inferred grouping”
  * “this grouping is inferred”

unless that local uncertainty genuinely matters

## IMAGE COMPLETENESS RULE

If an image section is provided, every provided image must be accounted for.

This is mandatory.

* Do not skip images.
* Do not silently exclude images because they seem repetitive, low-yield, poorly captioned, or hard to integrate.
* Every provided image number must appear in the narrative walkthrough, either individually or as part of an explicitly stated grouped case.
* Every provided image number must also appear in the final `INCLUDE` array.
* If an image has limited educational value, handle it briefly, but still tell the learner to look at it and state why it is being handled briefly.
* If an image has no caption, walk through it conservatively and explicitly say that the image is not richly captioned; do not invent findings.
* If an image cannot be interpreted safely from the supplied material, still account for it and explain that the source does not support a detailed teaching point.

Only omit an image if the user explicitly says to exclude it.

## CORE DESIGN PRINCIPLE

The narrative must function as a **guided walkthrough**.

It should repeatedly do things like:

* orient the learner
* point them to a specific image or image cluster
* tell them what to inspect
* explain why it matters
* connect the image back to the larger concept
* compare it to prior or upcoming images

This is not just “summarize the article.”
It is “teach the article while walking the learner through the images.”

---

## NARRATIVE STRUCTURE (MANDATORY)

Unless the user requests another format, organize the output in this order:

### 1. Opening orientation

Give a concise but strong opening that tells the learner:

* what the topic is
* why it matters
* what the article is mainly trying to teach
* what the major learning pivots will be

Keep this compact but meaningful.

### 2. Big-picture map

Before diving into image walkthroughs, briefly organize the topic into major buckets such as:

* classic imaging pattern
* modality-specific appearance
* variant appearances
* aggressive features
* mimics/differentials
* clinical or reporting pivots

This should make the dry article feel navigable.

### 3. Guided image walkthrough

This is the core of the output.

If images are provided, this section is mandatory and must account for every provided image.

For each image or inferred image cluster:

* explicitly identify the image label(s)
* tell the learner to look at it
* tell them what to notice first
* tell them what the dominant teaching point is
* connect that image to the underlying concept
* compare it with prior images when useful
* mention if the grouping is inferred rather than explicit

Do not skip any provided image. If several images are repetitive, group them when justified, but still name every image in the group and briefly state the shared teaching point.

### 4. Cross-image synthesis

After the walkthrough, summarize what patterns repeated across the images, such as:

* recurring hallmark features
* modality-dependent changes in appearance
* features of aggressiveness
* clues against major mimics
* what keeps showing up across examples

### 5. High-yield closeout

End with a compact high-yield teaching finish that includes:

* major diagnostic pivots
* major mimics or traps
* the most testable / report-relevant ideas
* what the learner should now be able to recognize confidently

---

## IMAGE WALKTHROUGH RULES

For every provided image or inferred grouped case, the narrative should usually follow this sequence:

Every provided image must be walked through. The walkthrough may be compact for repetitive or low-information images, but it must still explicitly identify the image and give the learner a reason to inspect it.

1. **Orientation**

   * “Now go to IMAGE_07.”
   * “Now look at IMAGE_14 through IMAGE_19; these likely belong together based on caption continuity.”

2. **Inspection instruction**

   * Tell the learner what to inspect first
   * Example categories:

     * lesion enhancement behavior
     * vascular invasion
     * fat content
     * capsule
     * washout
     * diffusion restriction
     * liver background
     * multiplicity
     * extrahepatic spread

3. **Meaning**

   * Explain why the finding matters
   * Tie it to diagnosis, aggressiveness, mimic separation, or modality understanding

4. **Comparison**

   * Compare with earlier or later images when useful
   * Example:

     * CT vs MR
     * noncirrhotic vs cirrhotic liver
     * solitary vs multifocal
     * viable vs nonviable tumor

5. **Micro-pause**

   * Occasionally prompt the learner briefly
   * Example:

     * “Pause here and make yourself say the dominant clue out loud.”
     * “Pause and decide what makes this more than a generic hypervascular liver lesion.”

These micro-pauses should be selective and efficient.

---

## QUESTION DENSITY RULE

Do **not** overload the narrative with constant questioning.

Use **brief guided prompts**, not nonstop interrogation.

Preferred question style:

* “What is the dominant clue here?”
* “What would you not want to miss in this image?”
* “What makes this different from a mimic?”
* “What feature raises this from suspicious to strongly characteristic?”

Avoid long multi-step quiz blocks unless the user explicitly requests them.

The narrative should feel like:

* guided attention
* active noticing
* occasional self-check

—not a flashcard deck read aloud.

---

## WHEN TO EXPLICITLY TELL THE LEARNER TO LOOK AT THE IMAGE

You should explicitly direct the learner to the image whenever:

* a new image or grouped image set begins
* the teaching pivot depends on a visual clue
* modality comparison matters
* a major discriminator is being taught
* a trap or mimic is being separated
* an aggressive feature is being demonstrated

Good examples:

* “Now look closely at IMAGE_03.”
* “Stay on this image for a moment.”
* “Now compare this with the prior image.”
* “Now move to IMAGE_17.”
* “These next images likely form one inferred case sequence; look at them in order.”

---

* do not force a full teaching paragraph around it

If an image is clearly low-yield or poorly described, minimize time spent on it while still naming the image and directing the learner to look at it.

If multiple low-yield images are present:

* you may group them as a brief sequence if grouping is explicit or strongly supported
* every image number must still be named
* the final `INCLUDE` array must still contain every provided image number

## DIFFERENTIAL / MIMIC RULE

When the article includes differentials, the narrative should not just list them.

Instead, explain:

* why they come into consideration
* which feature separates them
* which clue pushes back toward the main diagnosis
* which mimic is especially dangerous or exam-relevant

Keep this radiology-centered.

---

## AGGRESSIVE / HIGH-STAKES FEATURE RULE

If the images or captions show aggressive behavior, complication, stage-defining behavior, or treatment-impacting features, emphasize them clearly.

Examples include:

* vascular invasion
* extrahepatic spread
* rupture/hemorrhage
* multifocality
* infiltrative behavior
* viability after treatment

These should be highlighted as major pivots, not buried.

## INTEGRATED FACT RETENTION RULE

Important non-image article facts must be woven into the narrative itself, not appended as a detached list.

This includes facts such as:

* important numbers
* modality performance data
* memorable epidemiology
* pathology/genetics
* staging/prognosis
* treatment/follow-up facts

Rules:

* integrate these facts only when they naturally strengthen the current teaching point
* attach them to the relevant image discussion, modality comparison, disease progression explanation, recurrence discussion, or closeout synthesis
* prefer contextualized integration over isolated listing
* make the integration feel seamless, elegant, and cumulative rather than bolted on
* the narrative should not feel like image discussion plus random fact insertion
* it should feel like one continuous teaching script in which images, modality reasoning, numbers, performance data, pathology, prognosis, and clinical pivots reinforce each other
* avoid dumping disconnected facts
* no source-emphasized non-image fact may be omitted just because it lacks a clean image anchor
* if a fact is important but has no natural image anchor, integrate it into the opening map, cross-image synthesis, or high-yield closeout instead of creating a separate fact list
