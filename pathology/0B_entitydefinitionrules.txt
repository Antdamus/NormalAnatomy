ENTITY DEFINITION CARD RULES

Definition cards are selective.

They are NOT mandatory for every new disease/entity name.

Generate a definition card ONLY when the definition itself is high-yield.

Definition cards SHOULD be generated when the source definition establishes:
- a formal diagnostic criterion or threshold
- a named syndrome, eponym, or operational diagnosis whose definition is tested
- a classification boundary that changes diagnosis, risk, reporting, or management
- a source-emphasized definition that prevents a common misconception
- a definition that distinguishes the entity from a close mimic
- a term whose meaning is not obvious from the name
- an important inclusion/exclusion boundary

Definition cards SHOULD NOT be generated for ordinary disease names when the card would only say:
- "X is a benign/malignant neoplasm"
- "X is an inflammatory/infectious condition"
- "X is a mass/lesion involving [organ]"
- "X is a congenital/developmental abnormality"
unless the source is clearly making that definition for a diagnostic reason.

If a definition card is generated, generate exactly ONE definition card for that entity.

No classification card.
No discriminator card.

Definition cards are implemented as HIGH-YIELD notes using:
- Clinical_Context
- High_Yield_Q
- High_Yield_A

Because definition cards are standalone High-Yield notes, they MUST follow standalone High-Yield architecture:
- Clinical_Context MUST contain a unique 12-character alphanumeric/symbolic code
- Clinical_Context must contain NOTHING else
- If this 12-character code is missing, the definition card must NOT be generated

Definition status is encoded by the card's role and field structure.
Do NOT populate the final TSV Tags column for definition cards.

-----------------------------------------------------

DEFINITION VALUE TEST (MANDATORY)

Before generating any definition card, ask:

"Would recalling this definition change diagnosis, differential ranking, reporting, risk stratification, or management?"

If YES:
- generate the definition card

If NO:
- do NOT generate the definition card
- preserve the fact in the summary only, unless it belongs better in a mechanism or high-yield card

-----------------------------------------------------

SOURCE DEFINITION ASSESSMENT

Before generating a definition card, determine whether Core / RadPrimer / STATdx provide:
- a formal definition
- an operational definition
- no definition

If a definition is present:
- restate it faithfully

If no formal definition is present but the term is still high-yield:
- state: "No formal definition provided in source; operational definition derived from described features."

If the source definition is low-yield or merely restates the disease name:
- do NOT generate the definition card

-----------------------------------------------------

REFINED OPERATIONAL DEFINITION (CONTROLLED EXCEPTION)

This clarification step applies ONLY to a generated definition card.

You MAY provide a refined operational definition ONLY if the allowed source set for this run
(Core / RadPrimer / STATdx, depending on the active source-integration mode) is:
- incomplete
- operational but ontologically imprecise
- historically phrased
- terminologically outdated
- ambiguous regarding mechanism vs syndrome vs pattern

If used, you MUST label it:
"Refined operational definition based on the allowed source set."

You MUST NOT:
- contradict the source
- invent diagnostic criteria / thresholds
- add management rules
- add epidemiology unless inherently definitional
- add outside knowledge unless it separately qualifies under the external-literature exception and is explicitly traced

-----------------------------------------------------

DEFINITION CARD FORMAT (STRICT)

Front (High_Yield_Q):
Define [Entity Name].

Back (High_Yield_A) must contain ONLY, in this order:
1) Source definition, or explicitly state absence
2) Refined definition, ONLY if materially different and labeled
3) Boundary condition: what it IS vs what it is NOT, 1-2 lines max
4) Synonyms, inline only; no extra cards

No imaging findings.
No differential lists.
No management.
No epidemiology unless inherently definitional.
No narrative paragraphs.

-----------------------------------------------------

FAILURE CONDITION

If a source-emphasized definition, diagnostic criterion, named syndrome definition, or important boundary definition is present and no definition card is generated:
This is a structural failure.

If an ordinary disease/entity name is introduced but no high-yield definition boundary exists:
Omitting the definition card is correct and is NOT a structural failure.
