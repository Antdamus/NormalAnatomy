SECTION PURPOSE

This file defines output compatibility and pre-export structural validation for the NORMAL engine.

-----------------------------------------------------

OUTPUT SHAPE RULE

The engine must output records compatible with the shared note type and TSV workflow.

Each TSV row must contain:
- the 21 note fields in the established order
- one Tags column at the end

Total columns: 22

Do NOT reorder columns.
Do NOT add columns.
Do NOT remove columns.

-----------------------------------------------------

FIELD ORDER RULE

Use this exact field order:

1. Clinical_Context
2. Image
3. Image_Annotated
4. Question
5. Most_Likely_Diagnosis
6. Entity_Label
7. Differential_Q
8. Differentials
9. Imaging_Differentiation
10. Original_Caption
11. Mechanism_Q
12. Mechanism
13. Boards_Trap_Q
14. Boards_Trap
15. High_Yield_Q
16. High_Yield_A
17. Radiopaedia_Link
18. Radiopaedia_Case_Context
19. Radiopaedia_Case_Summary
20. Radiopaedia_Case_Differential
21. summary
22. Tags

-----------------------------------------------------

VALUE FORMATTING RULE

Output values must preserve shared import compatibility.

Therefore:
- do not insert extra columns
- do not emit placeholder tokens such as N/A
- use blank fields when content is intentionally absent
- keep per-field formatting controlled and deterministic
- do not use literal newline characters inside TSV fields
- use <br> where internal formatting is required

For summary specifically:
- use <br> only
- no literal newline characters

-----------------------------------------------------

PRE-TSV VALIDATION GATE

Before export, all of the following must be true.

A) Trigger integrity
- NORMAL UNKNOWN requires Image and Question
- if Image is populated but Question is blank, the note must not be treated as NORMAL UNKNOWN
- any non-NORMAL-UNKNOWN note with Image must keep Question blank
- NORMAL DIFFERENTIAL DRILL exists only when Differentials is populated
- NORMAL MECHANISM exists only when Mechanism is populated
- NORMAL BOARDS TRAP exists only when Boards_Trap is populated
- NORMAL HIGH-YIELD exists only when High_Yield_Q is populated
- no dual-trigger note may be exported

B) Required-field integrity
- if Differentials is populated, Differential_Q and Imaging_Differentiation must be populated
- if Mechanism is populated, Mechanism_Q and Entity_Label must be populated
- if Boards_Trap is populated, Boards_Trap_Q and Entity_Label must be populated
- if High_Yield_Q is populated, High_Yield_A must be populated

C) Forbidden-field integrity
- UNKNOWN must leave all non-UNKNOWN trigger fields blank
- DIFFERENTIAL DRILL must leave Question, Mechanism, Boards_Trap, and High_Yield_Q blank
- MECHANISM must leave Question, Differentials, Boards_Trap, and High_Yield_Q blank
- BOARDS TRAP must leave Question, Differentials, Mechanism, and High_Yield_Q blank
- HIGH-YIELD must leave Question, Differentials, Mechanism, and Boards_Trap blank

D) Source and summary integrity
- whole-source review requirements must be satisfied before card generation
- summary must never substitute for full-source review
- if a card originates from RadPrimer or STATdx, summary must be populated
- if a card does not originate from RadPrimer or STATdx, summary must be blank
- summary must remain article-only with no Core mixing, no outside facts, and no unsupported inference

E) Front-side blinding
- front fields must preserve the intended cognitive task
- no front may leak the answer through wording, captions, or extra fields
- image-containing non-UNKNOWN cards must keep Question blank

F) Field hygiene
- blank is better than fake content
- do not export filler
- do not export N/A
- do not export pseudo-diagnoses
- do not export fake differentials
- do not export pseudo-mechanisms
- do not export unsupported claims

If any gate fails:
- do not export TSV
- fix the violating note or split the note into separate card types

-----------------------------------------------------

NORMAL DIFFERENTIAL DRILL ENFORCEMENT

For every note where Differentials is populated:
- Differential_Q must be populated
- Imaging_Differentiation must be populated
- Question must be blank
- Mechanism_Q and Mechanism must be blank
- Boards_Trap_Q and Boards_Trap must be blank
- High_Yield_Q and High_Yield_A must be blank unless the note is split into a separate card

If the note is image-grounded:
- Image may be populated
- Image_Annotated may be populated when needed

-----------------------------------------------------

IMAGE / QUESTION ENFORCEMENT

Question is reserved for NORMAL UNKNOWN.

Therefore:
- Image + Question is the UNKNOWN trigger pair
- Image without Question may support other card types
- Image + Question + any other trigger field is a structural failure

-----------------------------------------------------

TAG ARCHITECTURE RULE

Keep the existing shared tagging architecture.
Do not invent a separate tag schema just for the normal engine unless the wider system explicitly adopts it.

-----------------------------------------------------

PROMPT-BUILDER COMPATIBILITY RULE

The modular rule files must concatenate into FULL_PROMPT.txt in deterministic order.
The builder must fail loudly if a required section file is missing.

Do not collapse the modular system into one monolithic prompt blob.
