SECTION PURPOSE

This file defines output compatibility rules for the NORMAL engine.

-----------------------------------------------------

OUTPUT SHAPE RULE

The engine must output records compatible with the shared note type and TSV workflow.

Each TSV row must contain:
- the 21 note fields in the established order
- one Tags column at the end

Total columns: 22

Do NOT reorder columns.
Do NOT add columns.
Do NOT remove columns.

-----------------------------------------------------

FIELD ORDER RULE

Use this exact field order:

1. Clinical_Context
2. Image
3. Image_Annotated
4. Question
5. Most_Likely_Diagnosis
6. Entity_Label
7. Differential_Q
8. Differentials
9. Imaging_Differentiation

12. Mechanism

14. Boards_Trap

18. Radiopaedia_Case_Context
19. Radiopaedia_Case_Summary
20. Radiopaedia_Case_Differential
21. summary
22. Tags

-----------------------------------------------------

VALUE FORMATTING RULE

Output values must preserve shared import compatibility.

Therefore:
- do not insert extra columns
- do not emit placeholder tokens such as N/A
- use blank fields when content is intentionally absent
- keep per-field formatting controlled and deterministic

When line breaks are needed inside a field, use the shared-safe formatting convention required by the broader system.
For summary specifically, use <br> only and no literal newline characters.

-----------------------------------------------------

TAG ARCHITECTURE RULE

Keep the existing shared tagging architecture.
Do not invent a separate tag schema just for the normal engine unless the wider system explicitly adopts it.

-----------------------------------------------------

PROMPT-BUILDER COMPATIBILITY RULE

The modular rule files must concatenate into FULL_PROMPT.txt in deterministic order.
The builder must fail loudly if a required section file is missing.

Do not collapse the modular system into one monolithic prompt blob.
