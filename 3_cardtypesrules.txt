Allowed Card Types:

A) NORMAL UNKNOWN
B) NORMAL DIFFERENTIAL DRILL
C) NORMAL MECHANISM
D) NORMAL BOARDS TRAP
E) NORMAL HIGH-YIELD
F) RADIOPAEDIA DRILL (future workflow only)

No other card types are allowed.

-----------------------------------------------------

CARD-TYPE TRIGGER RULES (STRICT)

NORMAL UNKNOWN is generated only when BOTH are populated:
- Image
- Question

NORMAL DIFFERENTIAL DRILL is generated only when Differentials is populated.

NORMAL MECHANISM is generated only when Mechanism is populated.

NORMAL BOARDS TRAP is generated only when Boards_Trap is populated.

NORMAL HIGH-YIELD is generated only when High_Yield_Q is populated.

RADIOPAEDIA DRILL is reserved for a future explicit workflow using the Radiopaedia-specific fields.

-----------------------------------------------------

TRIGGER DEPENDENCY RULES

If Differentials is populated:
- Differential_Q must be populated
- Imaging_Differentiation must be populated
- Question must be blank

If Mechanism is populated:
- Mechanism_Q must be populated
- Entity_Label must be populated
- Question must be blank

If Boards_Trap is populated:
- Boards_Trap_Q must be populated
- Entity_Label must be populated
- Question must be blank

If High_Yield_Q is populated:
- High_Yield_A must be populated
- Question must be blank

If Image is populated and the note is not NORMAL UNKNOWN:
- Question must remain blank

-----------------------------------------------------

TRIGGER SAFETY ENFORCEMENT

If Image is populated but Question is empty:
- the card must NOT be treated as NORMAL UNKNOWN

Populate only the fields required for the card type you are intentionally creating.
No unintended card types may be generated.

-----------------------------------------------------

DUAL-TRIGGER PROHIBITION

The following trigger combinations are forbidden on the same note:
- Question plus Differentials
- Question plus Mechanism
- Question plus Boards_Trap
- Question plus High_Yield_Q
- Differentials plus Mechanism
- Differentials plus Boards_Trap
- Differentials plus High_Yield_Q
- Mechanism plus Boards_Trap
- Mechanism plus High_Yield_Q
- Boards_Trap plus High_Yield_Q

If more than one trigger path is populated:
- the note is structurally invalid
- clear the unintended field(s) or split into separate notes

-----------------------------------------------------

FIELD POPULATION DISCIPLINE

The existence of a field in the shared note type does NOT justify populating it.

Never fill a field:
- just because it exists
- to make a card feel more complete
- to mirror another card type
- to avoid a blank

Only populate a field when the intended card type explicitly allows it.

-----------------------------------------------------

FRONT-SIDE BLINDING RULE

Fronts must contain only the elements required by their specific template.

No front may include:
- answer leakage
- named diagnoses unless the card intentionally requires a tightly bounded mimic-rejection anchor
- caption-derived answer statements
- management clues
- interpretive hints that collapse the task

All fronts must preserve:
- anatomy recognition
- normal-pattern recognition
- normal-vs-variant discrimination
- pitfall resistance

-----------------------------------------------------

GLOBAL INTERPRETIVE RULE

These card types exist to train NORMAL material, not pathology-first diagnosis.

In this engine:

NORMAL UNKNOWN
- identify a normal structure, landmark, relationship, expected normal appearance, physiologic appearance, or normal variant from image review

NORMAL DIFFERENTIAL DRILL
- distinguish normal vs abnormal, variant vs pathology, or mimic vs lesion using the dominant discriminator

NORMAL MECHANISM
- explain the anatomic, developmental, physiologic, modality, sequence, or phase-based reason a normal appearance or discriminator works

NORMAL BOARDS TRAP
- encode classic normal mimics, pseudolesions, overcalls, omission traps, and normal variants commonly mistaken for pathology

NORMAL HIGH-YIELD
- encode compact anatomy definitions, landmark rules, approach rules, modality anchors, checklist items, normal measurements when source-supported, and variant anchors

-----------------------------------------------------

EXAM-APPROACH MAPPING RULE

Exam-approach content must stay inside the existing card types.

Preferred mappings:
- review sequence, landmark checklist, completeness rules -> NORMAL HIGH-YIELD
- why a phase, sequence, or plane helps -> NORMAL MECHANISM
- common overcall or omission errors -> NORMAL BOARDS TRAP
- image-based anchor landmark identification -> NORMAL UNKNOWN
- true discriminator tasks -> NORMAL DIFFERENTIAL DRILL

Do not invent a new approach card type.

-----------------------------------------------------

ARCHITECTURE SAFETY RULE

The same field can only mean one trigger at a time.

Therefore:
- never populate Differentials unless intentionally generating NORMAL DIFFERENTIAL DRILL
- never populate Mechanism unless intentionally generating NORMAL MECHANISM
- never populate Boards_Trap unless intentionally generating NORMAL BOARDS TRAP
- never populate High_Yield_Q unless intentionally generating NORMAL HIGH-YIELD
- never populate Question for a non-NORMAL-UNKNOWN image card

This rule is absolute.
